The reconnection system includes:

  • Message Queue: Messages are queued when WebSocket is disconnected
  • Auto-Reconnect: Widget automatically attempts to reconnect when reopened
  • Queue Processing: Queued messages are sent once connection is established
  • Visual Feedback: Connection status and queue count shown in dev mode
  • Error Handling: Graceful handling of connection failures with retry logic
Note: This test page is running in development mode, so you'll see the connection status indicator. In production, the widget will work the same way but without the visual debug information.
